1. echo 'int main() { open("somefile", O_RDWR); }' > test.c
2. Run: cat test.c. What did you just write?
3. Command: gcc -c test.c -o test.o
4. Command: file test.o. What does file report?
5. Command: readelf -r test.o | grep open
6. What symbol type appears for open? Write as exact letters.
7. Mathematics: If compiler knew open address, relocations section would show TYPE = REL, not UND. UND = undefined means symbol resolution deferred to link time.
8. Command: objdump -d test.o | grep -A3 -B1 call
9. What 5 bytes follow as call opcode? Write in hex.
10. Mathematics: call opcode = 0xE8. 4-byte offset = 0x00000000. This placeholder will be filled later by linker.
2
11. Command: gcc test.o -o test
12. Command: ldd test. Which libraries does your binary need?
13. Command: objdump -d test | grep -A5 -B5 call
14. What address does call instruction target now? Write full address.
15. Command: readelf -a test | grep -A2 "Entry point"
16. What is as program entry point address?
17. Mathematics: Entry = 0x1050. PLT section starts here.
18. Explanation: Your call jumps to PLT trampoline, not directly to open().
